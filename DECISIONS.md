# Решения-отклонения

Здесь фиксируются только отклонения от ТЗ и выборы, требующие обоснования на защите.
Решения «по умолчанию» (рекомендованный стек, стандартные практики) не дублируются.

## 2026-07-20 — датасет: antonkozyriev/game-recommendations-on-steam вместо предложенного в ТЗ

ТЗ предлагает Steam Reviews Dataset, но допускает другой источник, если он лучше подходит.
Выбранный датасет закрывает все требования сразу: цена игры (фильтр по бюджету), теги и
описания (content-based), 41M взаимодействий с датами (временной сплит для honest
evaluation), бинарная оценка is_recommended, анонимные user ID, лицензия CC0.
Версия и sha256 файлов фиксируются в `data/raw/manifest.json` скриптом `ml/download_data.py`.

## 2026-07-20 — очередь задач: RQ, worker только в Docker

Из допущенных ТЗ вариантов (Celery / RQ / Dramatiq) выбран RQ: две редкие фоновые операции
(импорт, переобучение) не оправдывают сложность Celery. Ограничение: worker RQ требует
`fork()`, поэтому запускается только в Linux-контейнере (Docker Compose) — на Windows-хосте
напрямую не работает. Для локальной отладки обучения — отдельная синхронная команда без очереди.
