# Решения-отклонения

## 2026-07-20 — датасет: antonkozyriev/game-recommendations-on-steam вместо предложенного в ТЗ

ТЗ предлагает Steam Reviews Dataset, но допускает другой источник, если он лучше подходит.
Выбранный датасет закрывает все требования сразу: цена игры (фильтр по бюджету), теги и
описания (content-based), 41M взаимодействий с датами (временной сплит для honest
evaluation), бинарная оценка is_recommended, анонимные user ID, лицензия CC0.
Версия и sha256 файлов фиксируются в `data/raw/manifest.json` скриптом `ml/download_data.py`.

## 2026-07-21 — evaluation на подмножестве, а не на полных 41M взаимодействий

Сплит строится из сырых данных с фиксированными параметрами (топ-5000 игр с тегами,
50 000 пользователей с ≥5 отзывами, seed 42) вместо полного датасета. Причина: полный
прогон умножает время каждой итерации evaluation в десятки раз без изменения выводов
сравнения моделей, а дедлайн — 6 дней. Воспроизводимость сохранена: параметры, версия
датасета и sha256 исходников зафиксированы в `data/processed/split_manifest.json`.
Порог отсечения по времени — 2022-01-01 (test = последний год данных).

## 2026-07-22 — serving-модель: взвешенный гибрид item-item + популярность, не ALS

Открытый вопрос «implicit vs SVD» закрыт замером: wheel implicit под Python 3.13/Windows
существует, ALS обучен на дефолтных гиперпараметрах (64 фактора, без confidence-весов)
и уступил item-item — P@10 0.0134 против 0.0201; вывод ограничен конфигурацией «без
тюнинга», sparsity (медиана 1 отзыв) — вероятная причина.

В serving идёт гибрид `0.6·item-item + 0.4·time-decay-популярность`; веса выбраны
a priori до замеров, не тюнились (валидационного сплита нет — тюнить на test нельзя).
Топ-3 модели (item-item, hybrid, hybrid-artifact; P@10 0.0201–0.0210) статистически
неразличимы — **выбор гибрида обоснован cold start'ом** (популярностный член даёт выдачу
без истории), а не дельтой метрики. Формула evaluation и API одна; артефакт усечён до
топ-20 соседей на игру, поэтому в таблице отдельная строка `hybrid-artifact` — честная
метрика ровно того файла, что отвечает пользователям (усечение качество не ухудшило).
Артефакт — JSON без pickle (защита от несовместимости версий и небезопасной
десериализации); работающий сервер перечитывает его по mtime и сверяет версию с активной
ModelVersion, при рассинхроне уходит в baseline-fallback.

## 2026-07-20 — очередь задач: RQ, worker только в Docker

Из допущенных ТЗ вариантов (Celery / RQ / Dramatiq) выбран RQ: две редкие фоновые операции
(импорт, переобучение) не оправдывают сложность Celery. Ограничение: worker RQ требует
`fork()`, поэтому запускается только в Linux-контейнере (Docker Compose) — на Windows-хосте
напрямую не работает. Для локальной отладки обучения — отдельная синхронная команда без очереди.
